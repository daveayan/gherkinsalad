Welcome to Gherkin Salad

Project Setup

Use the maven archetype using the following command 

mvn archetype:generate -DarchetypeGroupId=daveayan -DarchetypeArtifactId=gherkinsalad.sample-archetype -DarchetypeRepository=https://github.com/daveayan/_mavenrepository
cd to_your_project
mvn clean test
OR Use the following instructions
  • Create a maven project
  • Add the following dependency in the pom.xml 
    <dependency>
	<groupId>daveayan</groupId>
	<artifactId>gherkinsalad</artifactId>
	<version>1.0</version>
</dependency>

    <repositories>
	<repository>
		<id>daveayan-releases</id>
		<url>https://github.com/daveayan/_mavenrepository/raw/master/releases</url>
	</repository>
</repositories>
<pluginRepositories>
	<pluginRepository>
		<id>daveayan-plugins</id>
		<url>https://github.com/daveayan/_mavenrepository/raw/master/releases</url>
		</pluginRepository>
</pluginRepositories>
  • Put your gherkins in src/test/resources/cucumber/gherkins
  • Put stepdefs in src/test/java/cucumber/stepdefs
  • Run mvn clean test

Writing your step definitions and page objects

Create your stepdef and page object java files and let them extend BaseBrowserElement.java 


public class SomeStep extends BaseBrowserElement {}

public class SomePageObject extends BaseBrowserElement {}

Working with Browser

To launch a browser use the launch_browser method. Currently supported: Firefox, chrome. The browser that is launched is available to any java class that extends BaseBrowserElement. 


super.launch_browser("Firefox");
OR
super.launch_browser("Chrome");

To close the currently open browser 


super.close_browser();

To navigate to a site 


super.goto_url("http://www.gherkinsalad.com");

Taking Screenshots

To take a screenshot of the current screen on the browser window use the takeScreenshot method. The screenshot is taken as a png file in the target/screenshots folder and saved as "screenshot_###.png". The ### is incremented every time next screenshot is taken. 


super.takeScreenshot();

Selecting elements

gherkinsalad framework relies on the selenium selectors 


By.id("someid");
By.name("somename");
By.className("someclassname");
By.cssSelector("somecssselector");
By.linkText("somelinktext");
By.partialLinkText("somepartiallinktext");
By.tagName("sometagname");
By.xpath("somexpath");

Clicking elements


Clickable some_link = Link.find(By...);
some_link.click_if_enabled();

Text Entry Elements


TextEnterable some_text_field = TextField.find(By...);
some_text_field.enter_text_if_enabled(""); // pass in some text

Dropdowns


Selectable some_dropdown = Dropdown.find(By...);
some_dropdown.select_option_if_enabled(""); // pass in option text

Property File Usage

The framework will look for the property file in resources/system folder. It will first look for a property file named __userid__.ghksalad.properties where __userid__ is the login userid of the current user. If this file is found the framework will use it otherwise it will look for the property file default.ghksalad.properties. The contents of the property file has to have at least the following: 


seconds.wait.after.each.step=0
seconds.timeout=10
seconds.poll.interval=1

browser.chrome.enabled=true
browser.firefox.enabled=true
browser.ie.enabled=true
